
配置文件目錄
vi /etc/mysql/my.cnf

登入
mysql -h 192.168.1.100 -P 3306 -u root -p

显示所有數據庫：
SHOW DATABASES;

显示所有表：
SHOW TABLES;

查看錶結構：
DESCRIBE 表名;

創建用戶：
CREATE USER '用戶名'@'主機' IDENTIFIED BY '密碼';

授予權限：
GRANT ALL PRIVILEGES ON *.* TO '用戶名'@'主機';

刷新權限：
FLUSH PRIVILEGES;

查看當前使用的數據庫：
SELECT DATABASE();

查看當前會話的 SQL 模式:
SELECT @@sql_mode;

設置當前會話的 SQL 模式:
SET @@sql_mode = 'STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_ENGINE_SUBSTITUTION';

查看默認隔離級別:
select @@transaction_isolation;

------------------ SHOW ----------------

查看當前正在執行的查詢:
SHOW PROCESSLIST;

查看MySQL本次啟動后的運行時間(單位：秒):
SHOW STATUS LIKE 'uptime';

查看select語句的執行數:
SHOW GLOBAL STATUS LIKE 'com_select';

查看insert語句的執行數:
SHOW GLOBAL STATUS LIKE 'com_insert';

查看update語句的執行數:
SHOW GLOBAL STATUS LIKE 'com_update';

查看delete語句的執行數:
SHOW GLOBAL STATUS LIKE 'com_delete';

查看試圖連接到MySQL的連接數:
SHOW STATUS LIKE 'connections';

查看線程緩存內的線程的數量:
SHOW STATUS LIKE 'threads_cached';

查看當前打開的連接的數量:
SHOW STATUS LIKE 'threads_connected';

查看創建用來處理連接的線程數:
SHOW STATUS LIKE 'threads_created';

查看激活的線程數:
SHOW STATUS LIKE 'threads_running';

查看立即獲得的表的鎖的次數:
SHOW STATUS LIKE 'table_locks_immediate';

查看不能立即獲得的表的鎖的次數:
SHOW STATUS LIKE 'table_locks_waited';

查看創建時間超過slow_launch_time秒的線程數:
SHOW STATUS LIKE 'slow_launch_threads';

查看查詢時間超過long_query_time秒的查詢的個數:
SHOW STATUS LIKE 'slow_queries';

查看與 InnoDB 行操作相關的狀態變量:
SHOW STATUS LIKE 'innodb_rows_%';

查看與 MySQL 上一次查詢相關的成本:
SHOW STATUS LIKE 'last_query_cost';

查詢慢查詢日誌
SHOW global status LIKE '%Slow_queries%';


------------------啟用慢查詢日誌 重啟就沒了-----------------

-- 查看是否啟用了慢查詢日誌
SHOW global variables LIKE '%slow%';
SHOW VARIABLES LIKE 'slow_query_log';
SHOW VARIABLES LIKE 'slow_query_log%';

-- 全局啟用慢查詢日誌
set global slow_query_log = on;

-- 查看當前的慢查詢時間閾值（以秒為單位）
SHOW VARIABLES LIKE '%long_query_time%';

-- 設置慢查詢時間閾值為 1 秒
set global long_query_time = 1;
set long_query_time = 1;

-- 全局關閉慢查詢日誌
set global slow_query_log = 'off';

------------------啟用慢查詢日誌 永久方式-----------------


編輯 MySQL 配置文件（通常是 my.cnf 或 my.ini），添加或修改以下參數：
[mysqld]
slow_query_log = NO #開啟慢查詢日誌
slow_query_log_file = /var/lib/mysql-slow.log #慢查詢日誌目錄和文件訊息
long_query_time = 2  # 記錄執行時間超過2秒的查詢
log_output=FILE

重新啟動 MySQL 服務以使配置生效：
sudo service mysql restart

如果慢查詢日誌已啟用，你可以直接查看日誌文件。使用以下命令查看最新的慢查詢：
tail -n 100 /path/to/your/mysql-slow.log


slow_query_log = OFF #關閉慢查詢日誌
